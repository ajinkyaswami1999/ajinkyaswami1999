# Security Policy: Jurassic QA Lab Containment Protocol

## Supported Versions

Only the latest version of the Jurassic QA Lab automation suites and configurations are supported. Security patches are applied directly to the main branch.

| Version | Supported |
| ------- | --------- |
| > 2.x   | ✅ Yes    |
| < 2.x   | ❌ No     |

## Reporting a Vulnerability

> [!WARNING]
> Security breaches in paddock automation or laboratory networks must be reported immediately to prevent containment failure.

If you discover a security vulnerability (such as a memory leak, API token exposure, or logic exploit in the test suites), please follow the protocols below:

1. **Do not open a public issue.**
2. Report the vulnerability directly to the Head of QA Systems Automation by email at **ajinkyaswami1999@gmail.com**.
3. We will acknowledge receipt of your report within 48 hours and provide a timeline for patch deployment.
4. Once the patch is deployed to the main branch, credit will be awarded in the achievements registry.
