# Contributing to Jurassic QA Lab

We welcome contributions to strengthen our park's automated systems! Please read these guidelines to ensure containment protocols are maintained.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](file:///c:/AJINKYA/Personal/Projects/Git%20profile/CODE_OF_CONDUCT.md).

## How Can I Contribute?

### 1. Reporting Bugs
- Search existing issues before opening a new one.
- Describe the bug clearly: specify the automation framework (Selenium, Appium, Playwright), execution log, and environment details.
- Provide a minimal reproducible test script.

### 2. Suggesting Paddock Upgrades
- Open an issue explaining the proposed optimization or new tool integration.
- Detail the benefits to the overall automated containment system.

### 3. Pull Requests
- Fork the repository.
- Create a feature branch matching the paddock sector (e.g., `feature/paddock-1-fences`).
- Write clean, documented Python/TypeScript code.
- Ensure all test suites pass.
- Submit a Pull Request targeting the `main` branch.

## Coding Style Guidelines

- Keep automation scripts modular and reusable.
- Follow PEP 8 guidelines for Python scripts.
- Use explicit page object models where applicable.
- Add comments explaining complex test conditions.
